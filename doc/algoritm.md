

## Общий принцип работы

Пользователь вводит 4-значный код (в шестнадцатеричной системе), приложение преобразует его по определённому алгоритму и выводит результат в виде валидного сервисного ключа.

## Пошаговая логика

1.  **Ввод данных**  
    Пользователь вводит 4 символа в поле `EditText` (id: `syotto`).

2.  **Проверка длины**  
    Если длина ≠ 4 символа → вывод: «Unlock code length 4 digit!».

3.  **Проверка шестнадцатеричности**  
    Попытка преобразовать ввод в число по основанию 16 (`Integer.parseInt(..., 16)`).  
    Если ошибка → вывод: «Invalid unlock code!».

4.  **Преобразование кода**  
    Введённое число обрабатывается функцией `getServiceCode()`, которая:
    - Применяет битовые операции (AND, сдвиги) к входному коду.
    - Собирает новый код из отдельных битовых фрагментов.
    - Выполняет XOR с константой 43605.

5.  **Форматирование вывода**  
    Результат преобразуется в шестнадцатеричную строку и дополняется нулями до 4 символов:
    - Если результат > 4095 → просто `XXXX is valid key code`.
    - Если 256 ≤ результат ≤ 4095 → `0XXX is valid key code`.
    - Если 16 ≤ результат ≤ 255 → `00XX is valid key code`.
    - Если результат ≤ 15 → `000X is valid key code`.

---

## Детали алгоритма преобразования

Функция `getServiceCode(int unlockCode)` выполняет:

```
keyCode = ((unlockCode & 3) << 14)
          + ((unlockCode & 60) << 4)
          + ((unlockCode & 960) << 4)
          + ((unlockCode & 15360) >> 8)
          + ((49152 & unlockCode) >> 14)
          ⊕ 43605
```

Где:
- `&` — битовое И,
- `<<` — сдвиг влево,
- `>>` — сдвиг вправо,
- `⊕` — XOR (исключающее ИЛИ).
