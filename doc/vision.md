## Концепция

В этом документе описывается техническое видение приложения METSO HP300 code generator, основанное на идее, изложенной в "idea.md`.

## Технологии

- **Язык программирования**: Python 3.x  
  - Широко доступный, простой в разработке и обслуживании.
  - Достаточно для небольшой настольной утилиты без дополнительных сложностей.

- **Менеджер пакетов**: uv  
  - Используется для управления зависимостями и изолированными окружениями Python.
  - Легкий и быстрый инструмент, подходящий для небольших проектов.

- **GUI toolkit**: Tkinter (стандартная библиотека Python)  
  - Поставляется с Python без дополнительных зависимостей.
  - Подходит для очень простого окна с полем ввода, кнопкой и меткой результата.
  - Минимизирует объем проекта и упрощает его запуск на обычном ПК с Windows.

- ** Целевая платформа**: Рабочий стол Windows  
  - Основной вариант использования: запуск на ПК инженера /техника с Windows рядом с дробилкой.
  - На данном этапе нет требований к Linux/macOS.

- ** Хранение данных**: нет для MVP.  
  - Приложение принимает только входной код и вычисляет выходной код.
  - В первой версии нет базы данных, нет постоянных журналов, нет файлов конфигурации.

- ** Подход к развертыванию**  
  - ** Этап 1 (разработка/MVP) **: запуск непосредственно из исходного кода с установкой системного Python (например, `python main.py`).
  - ** Этап 2 (необязательный) **: если это необходимо для реального использования, упакуйте приложение в один исполняемый файл (например, с помощью PyInstaller), чтобы пользователь мог запускать приложение без установки Python.

- ** Запуск через uv**  
  - Предполагается, что `uv` уже установлен в системе.  
  - **Вариант 1 (проектное окружение)**:  
    - Выполнить один раз в каталоге проекта: `uv venv`  
    - Запуск приложения: `.\.venv\Scripts\python.exe main.py`  
  - **Вариант 2 (однократный запуск)**:  
    - Запуск без ручного создания окружения: `uv run python main.py`

## Принципы разработки

- **KISS (Keep It Simple, Stupid)**    
  - Мы выбираем максимально простое решение, которое подходит для текущей цели.
  - Мы избегаем ненужных абстракций, шаблонов и чрезмерной инженерии.

- **MVP-first**  
  - Первая цель - это рабочая минимальная версия: ввод кода → расчет → отображение результата.
  - Все дополнительные функции (журналы, настройки, поддержка нескольких моделей) отложены до тех пор, пока основной поток не заработает надежно.

- **Единая ответственность для каждого компонента**  
  - Каждая часть кода должна выполнять одну задачу: пользовательский интерфейс, логику вычислений, (позднее) ведение журнала и т.д.
  - Благодаря этому код остается небольшим, читаемым и легко изменяемым.

- **Explicit over implicit**   
  - Предпочтение отдается ясному, прямолинейному коду, а не “хитроумным” трюкам.
  - Алгоритм расчета должен быть реализован таким образом, чтобы его было легко понять и проверить.

- **Локальный, автономный, без внешних зависимостей**  
  - Приложение работает полностью автономно на локальном компьютере.
  - В MVP мы избегаем использования внешних сервисов и громоздких сторонних библиотек.
